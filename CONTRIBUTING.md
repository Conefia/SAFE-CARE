# Contributing

Contributions that improve clarity, reproducibility, accessibility, evaluation rigor, or cross-domain applicability are welcome.

## Before opening a pull request

- Do not include personal health information, proprietary client material, private prompts, credentials, or production traces.
- Use synthetic or fully de-identified examples.
- Explain the safety problem the change addresses and add or update a regression scenario when applicable.
- Keep high-risk clinical claims supported by authoritative sources and clearly distinguish engineering evaluation from clinical validation.
- Update the changelog and relevant machine-readable template.

## Review priorities

Safety impact, reproducibility, evidence quality, privacy, accessibility, and operational usefulness take priority over stylistic preferences.
